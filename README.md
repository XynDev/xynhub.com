# XYN Engineering Ecosystem

An advanced architecture for low-latency syntactic computing interfaces. Built natively with React (Vite) and orchestrated within an NPM Workspace monorepo environment to ensure pure performance and modular expansion.

## 🚀 Features

- **Monolithic Performance**: Built leveraging highly optimized React components powered by Vite.
- **The "Static Elegance" Design System**: Emphasizing high-density, bento-grid structures layered seamlessly to avoid visual fragmentation.
- **Tokenized Theming Contexts**: Architected utilizing Tailwind CSS v4 design tokens modeled exactly after pristine aesthetic rules (`UI_RULES.md`).
- **Nodal Mock Ecosystem**: Powered by deeply integrated structural JSON for frictionless transition to robust backend states.

## 🛠️ Tech Stack

- **Framework**: [Vite + React](https://vitejs.dev/) (Client-side SAP)
- **Monorepo Architecture**: NPM Workspaces
- **Styling**: [Tailwind CSS v4](https://tailwindcss.com/)
- **Language**: [TypeScript](https://www.typescriptlang.org/)
- **Icons**: [Material Symbols](https://fonts.google.com/icons)

## 🏁 Getting Started

### Prerequisites

- Node.js 18+
- npm v9+

### Installation & Initialization

1. **Clone the nodal network**
   ```bash
   git clone https://github.com/ilramdhan/xynhub.com.git
   cd xynhub.com
   ```

2. **Establish localized dependencies**
   ```bash
   # Executes a global installation mapped across all workspaces
   npm install
   ```

3. **Initialize Core Systems (Development Server)**
   ```bash
   # Navigate to the core web layer and boot the node
   cd apps/web
   npm run dev
   ```
   Open [http://localhost:5173](http://localhost:5173) to interface with the site.

## 📂 Project Structure

Operating via an NPM Monorepo architecture to maximize horizontal scalability:

```
xynhub.com/
├── apps/                         # Nodal Interfaces
│   └── web/                      # Core Web Application
│       ├── src/
│       │   ├── components/       # Atomic/Modular Elements
│       │   │   ├── layout/       # Structural Elements (Header, Footer)
│       │   │   └── ui/           # Static Elements (BentoCard, Button, Badge)
│       │   ├── data/             # JSON Mock Architecture
│       │   └── lib/              # Ecosystem Utilities
├── docs/                         # Extended Ecosystem Information
├── UI_RULES.md                   # Strict implementation guidelines for monolithic design
├── CONTRIBUTING.md               # Node expansion standards
└── package.json                  # Workspace definitions
```
